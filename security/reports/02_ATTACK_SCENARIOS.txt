  NEUROMONITOR — ESCENARIOS DE ATAQUE SIMULADOS

Perspectiva: atacante con motivación reconnaissance, DoS, escalada local.

Leyenda:
  [CONFIRMADO]  — reproducible con diseño/código actual
  [PROBABLE]    — requiere condición de despliegue documentada
  [LATENTE]     — depende de código divergente o fix futuro incompleto
  [MITIGADO]    — control reduce viabilidad significativamente

ESC-01 — RECONOCIMIENTO EN LOCALHOST (API habilitada)
--------------------------------------------------------------------------------
Actor        : Malware local, extensión navegador, script usuario
Prerrequisito: NEUROMONITOR_ENABLE_API=true [PROBABLE — opt-in requerido]
Vector       : GET http://127.0.0.1:8765/metrics/snapshot
Técnicas     :
  1. Escanear puerto 8765 en loopback
  2. curl sin autenticación → SystemSnapshot JSON completo
  3. Extraer mountpoints, hostname, GPU, RAM total
Objetivo     : Fingerprinting host de alto valor
Impacto      : ALTO en localhost; MEDIO en LAN (bind remoto requiere ALLOW_REMOTE)
Detección    : Sin audit log HTTP hoy
Mitigación   : SEC-001 auth token; SEC-004 disclosure redacted
Viabilidad   : [PROBABLE] — mitigado vs LAN por localhost default [MITIGADO parcial]

--------------------------------------------------------------------------------
ESC-02 — FLOOD DE SNAPSHOTS (Auto-DoS por observador)
--------------------------------------------------------------------------------
Actor        : Script local o remoto (si ALLOW_REMOTE)
Prerrequisito: API accesible sin rate limit [CONFIRMADO]
Vector       : while true; do curl -s http://127.0.0.1:8765/metrics/snapshot; done
Técnicas     :
  1. 20-50 req/s × cpu_percent(interval=0.05) ≈ 1-2.5s CPU bloqueada/s
  2. Cada request: 4 collectors + validación Pydantic
Objetivo     : Degradar máquina más que carga real del sistema
Impacto      : ALTO
Mitigación   : SEC-008 rate limit 2 rps/(IP+X-API-Token) + cache TTL [DISEÑO P1]
               Ver 07_SEC-008_RATE_LIMITING_SPEC.txt
Viabilidad   : [CONFIRMADO pre-mitigación] → [MITIGADO parcial post-implementación]

Simulación:
  concurrent=32, duration=60s
  → Saturación núcleo Python + psutil overhead

ESC-03 — WEBSOCKET FAN-OUT (árbol src/)
--------------------------------------------------------------------------------
Actor        : Cliente automatizado
Prerrequisito: uvicorn sobre src/neuromonitor/api/app.py [LATENTE]
Vector       : Múltiples WS /ws/metrics
Técnicas     :
  1. stream_snapshots() no existe → AttributeError (DoS funcional hoy)
  2. Si se implementa sin broadcaster: N conexiones = N loops psutil
Objetivo     : Multiplicar carga de observabilidad
Impacto      : ALTO (futuro); MEDIO (error actual)
Mitigación   : SEC-009 broadcaster único; usar paquete neuromonitor/ sin WS
Viabilidad   : [LATENTE] en paquete [CONFIRMADO roto] en src/

ESC-04 — ABUSO DEL BRIDGE pywebview
--------------------------------------------------------------------------------
Actor        : Malware local, consola devtools (si debug), XSS previo
Prerrequisito: NeuroMonitor desktop en ejecución [CONFIRMADO]
Vector       : pywebview.api.*
Técnicas     :
  1. get_history(9999999) → cap 500 [MITIGADO] retorna ≤500
  2. update_settings({poll_interval_ms: 100}) → max agresividad [CONFIRMADO]
  3. update_settings({app_name: "⚠ Falso Admin"}) → spoofing título [CONFIRMADO]
Objetivo     : DoS local, phishing UI, max polling
Impacto      : MEDIO (reducido vs auditoría previa por cap history)
Mitigación   : SEC-010, SEC-011, SEC-026 validación app_name
Viabilidad   : [CONFIRMADO]

ESC-05 — SECUESTRO DE MÉTRICAS (onNewSnapshot hijack)
--------------------------------------------------------------------------------
Actor        : Script en contexto WebView
Prerrequisito: Ejecución JS en file:// [PROBABLE con XSS/CDN compromise]
Vector       : window.onNewSnapshot = (fake) => { ... }
Técnicas     :
  1. Reemplazar callback antes del push evaluate_js
  2. Mostrar CPU 20% mientras malware consume 95%
Objetivo     : Ocultar actividad maliciosa al operador
Impacto      : MEDIO-ALTO (integridad operativa)
Mitigación   : SEC-013 HMAC; SEC-019 eliminar global; SEC-018 IPC nativo
Viabilidad   : [PROBABLE]

--------------------------------------------------------------------------------
ESC-06 — INYECCIÓN VÍA evaluate_js
--------------------------------------------------------------------------------
Actor        : Atacante controlando campos snapshot (admin local)
Vector       : desktop.py f-string + json.dumps(payload)
Análisis     :
  - hostname malicioso con comillas → json.dumps escapa correctamente
  - U+2028/U+2029 → Python json.dumps seguro en 3.11+
  - Cambio futuro a concatenación manual → RCE posible
Objetivo     : RCE en WebView
Impacto      : BAJO hoy; ALTO si regresión en serialización
Mitigación   : SEC-018 eliminar evaluate_js
Viabilidad   : [CONFIRMADO mitigado] con json.dumps actual

ESC-08 — CORS MISCONFIGURATION (src/neuromonitor/)
--------------------------------------------------------------------------------
Actor        : Sitio web malicioso + API localhost sin auth
Prerrequisito: Parche cors_origins=["*"] en Settings [LATENTE]
Vector       : fetch('http://127.0.0.1:8765/metrics/snapshot') desde HTTPS
Técnicas     :
  1. src/api/app.py ya tiene allow_credentials=True
  2. Sin auth, CORS restrictivo no basta en todos los browsers
Objetivo     : Exfiltrar métricas cross-origin
Impacto      : ALTO en escenario parche apresurado
Mitigación   : SEC-021 unificar código; SEC-001 auth; nunca * + credentials
Viabilidad   : [LATENTE]
