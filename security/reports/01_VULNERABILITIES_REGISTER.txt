CATEGORÍA A — AUTENTICACIÓN Y CONTROL DE ACCESO
--------------------------------------------------------------------------------

SEC-001 | API REST sin autenticación ni autorización
  Riesgo   : ALTO
  CVSS     : 7.5 (AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:N/A:N)
  Estado   : PARCIAL (contención P0) / ABIERTO (auth)
  Componente: neuromonitor/api/routes/metrics.py, health.py
  Evidencia:
    - GET /metrics/snapshot accesible sin credenciales cuando enable_api=true
    - enable_api=false por defecto; run_api_server() rechaza arranque sin opt-in
    - host restringido a 127.0.0.1 salvo ALLOW_REMOTE explícito
    - No hay middleware auth, API keys, mTLS ni IP allowlist
  Impacto  : Cualquier proceso en localhost (o LAN si ALLOW_REMOTE) lee telemetría.

SEC-002 | Bridge pywebview (RPC) accesible sin restricción desde JS
  Riesgo   : MEDIO (ALTO si XSS o HTML malicioso)
  CVSS     : 6.5 (AV:L/AC:L/PR:L/UI:R/S:U/C:H/I:L/A:L)
  Estado   : ABIERTO
  Componente: neuromonitor/bridge/pywebview_bridge.py, desktop.py
  Evidencia:
    - Métodos: get_current_snapshot, get_history, update_settings
    - pywebview expone js_api a todo el contexto JavaScript
    - Allowlist parcial en update_settings (_SETTINGS_ALLOWLIST)
  Impacto  : Lectura historial, cambio poll_interval_ms/enable_gpu/app_name.

SEC-003 | FastAPI OpenAPI (/docs, /redoc) expuesto por defecto
  Riesgo   : MEDIO
  CVSS     : 5.3
  Estado   : ABIERTO
  Componente: neuromonitor/api/app.py
  Evidencia: create_app() sin docs_url=None, redoc_url=None
  Impacto  : Reconocimiento automatizado de endpoints y esquemas.

CATEGORÍA C — DENEGACIÓN DE SERVICIO Y ABUSO DE RECURSOS
--------------------------------------------------------------------------------

SEC-008 | Snapshot REST sin rate limiting; psutil bloqueante
  Riesgo   : ALTO (API habilitada) / MEDIO (local)
  CVSS     : 7.5 (A:H)
  Estado   : Mitigación en Diseño (P1)
  Componente: api/routes/metrics.py, collectors/cpu.py
  Evidencia:
    - Cada GET /metrics/snapshot → capture_snapshot() completo [PRE-mitigación]
    - cpu_percent(interval=0.05) bloquea ~50ms
    - Sin throttling, cache TTL ni semáforo en código actual
    - LAB_09 TC-QA-100 FAIL; TC-QA-103 ratio cpu_calls/request ≈ 1.0
  Diseño   : security/reports/07_SEC-008_RATE_LIMITING_SPEC.txt
    - Guard rate_limit: 2 req/s por (IP + X-API-Token)
    - Store in-memory acotado: ring timestamps K=2 + LRU max_keys=4096
    - HTTP 429 opaco: ERR_RATE_LIMITED (sin fuga stack uvicorn)
  Impacto  : Auto-DoS por observador; agotamiento CPU.
  Nota     : Riesgo ALTO persiste hasta implementación y ACC-008-* PASS en CI.

SEC-009 | WebSocket: diseño fan-out / stream_snapshots ausente
  Riesgo   : ALTO (cuando WS activo)
  CVSS     : 7.5
  Estado   : PARCIAL
  Componente:
    - neuromonitor/api/app.py — WS NO registrado (MITIGADO en paquete)
    - src/neuromonitor/api/app.py — WS ACTIVO (ABIERTO)
    - stream.py — stream_snapshots() no existe en MonitorService
  Evidencia:
    - Paquete: comentario SEC-009, router WS excluido
    - src/: include_router(ws_router) + metrics_stream llama stream_snapshots()
  Impacto  : AttributeError hoy en src/; fan-out N×psutil si se implementa mal.

SEC-010 | get_history(limit) y --history-size sin tope superior coherente
  Riesgo   : MEDIO
  CVSS     : 5.5
  Estado   : PARCIAL
  Componente: pywebview_bridge.py, desktop.py
  Evidencia:
    + get_history: min(limit, 500) — MITIGADO OOM vía RPC
    - desktop --history-size: sin max (p.ej. 10_000_000 en RAM del buffer)
  Impacto  : OOM local vía CLI args; RPC acotado a 500 items.

SEC-011 | poll_interval_ms mínimo 100ms permite sobrecarga auto-infligida
  Riesgo   : MEDIO
  CVSS     : 4.0
  Estado   : ABIERTO
  Componente: config/settings.py (ge=100), bridge update_settings
  Evidencia: RPC puede fijar 100ms → 10 polls/s × coste psutil
  Impacto  : Degradación rendimiento del host monitorizado.

SEC-012 | WebSocket slow consumer sin backpressure
  Riesgo   : BAJO-MEDIO
  CVSS     : 4.5
  Estado   : ABIERTO (latente; WS off en paquete)
  Componente: src/neuromonitor/api/websocket/stream.py
  Evidencia: send_text en loop sin timeout ni drop policy
  Impacto  : Buffers crecientes si cliente no lee.
